# 不要盯着Agent干活

## AI 增强版 (AI-Enhanced Version)

这半年我和 coding agent 高强度协作下来，最大的心得就一句：

**不要盯着它干活。**

真的，不夸张。
我前段时间甚至被 AI 用得有点疲劳，后来一想，这种感觉特别像我刚开始带实习生的时候。

AI 确实能解放生产力。
但问题是，它的生产力很多时候是**盲目的、混乱的、不可预测的**。

你会发现，很多时候不是它能力不行，也不只是上下文窗口的问题。
更大的问题是：

**产品和开发，本来就是无数微小决策的累积。**

如果你只是随口说一句：
“帮我搞个报表。”
那它埋头一顿干，最后做成啥样，你可能都能接受。
因为 you 本来就不在乎细节，模糊需求对应模糊结果，这没问题。

但一旦到了真正的生产环境，事情就变了。

只要你有具体规范、具体约束、具体要求，
你就必须让 agent 知道它到底在干什么。

不然它就会一路带着自己的假设往前冲。
一个小误解，叠一个小误解，最后整个方向都歪掉。

所以我现在越来越觉得：

**很多人所谓的 vibe coding，本质上其实是一种微管理。**

你表面上是在“自然地和 AI 沟通”，
实际上是在不断盯着它、纠正它、替它做细碎决策。

而这件事为什么特别累？

因为管理人，至少对方节奏没那么快。
但 agent 是一个执行速度极高的东西，
你一旦开始微管理它，你就会进入一种高频、碎片化、持续不断的决策状态。

更烦的是，
你做决策之前，还得先看懂它到底写了什么玩意。

这才是很多人用了 AI 之后反而更累的原因。
不是没提效，
而是你从“自己干活”变成了“高强度盯一个不会停的实习生干活”。

后来我慢慢意识到，
问题根本不在于盯住它的每一步决策对不对。

真正该盯的，是：

**它为什么会做出这样的决策。**
**有没有机制让它自己得到反馈。**
**它做错之后，能不能自己校正。**

说白了，
重点不是盯执行过程，
重点是设计它的约束、链条和反馈机制。

这件事其实和带人很像。

你不是盯着一个人每一步都别出错，
而是要让他知道目标、边界、标准，以及出了问题怎么自己修。

现在很多 coding agent 其实已经很强了，
强到可以自己跑很久，自己查资料，自己测试，自己拿反馈，自己修很多具体问题。

只有这样，AI 才是真的在解放生产力。
而不是把你变成一个永远坐在旁边、随时准备接锅的监工。

所以我现在越来越觉得：

**和 coding agent 协作，重点不是“盯着它干活”，而是“让它在不被盯着的情况下也能干活”。**

这可能才是 AI 真正能进入生产环境的前提。

有人看的话，我下一篇细说：
**我具体做了哪些事，才敢不盯着 AI 干活。**

## 原始版本 (Original Version)

作为一个每月token费用爆炸的重度开发者，这半年我能总结的和coding agent协作的最重要的诀窍就是，不要看着它干活。。。

在过去的半年中，我重度使用ai，有段时间甚至感觉异常疲劳，这让我回忆起来了我最开始带实习生的日子，也让我开始重新反思我们和ai的关系。

实际上ai确实能解放生产力，但是它的生产力是盲目的，混乱的和不可预测的。你会发现跟ai的交互中，越做越歪，越说越听不懂啥的，但这在很大程度上不光是因为所谓的上下文窗口啊啥的这种技术问题，更是因为我们口中所谓的产品是无数微小决策的累积。

自然语言的模糊性有其好处。比如当你并不在乎数据看版具体长啥样的时候，你一句弄个报表，下面的agent就埋头猛干去了。因为你真的不在乎，所以一切结果对你来说都可以接受，那么自然没有任何问题，在很多时候能带来让人惊喜的结果。

但在生产环境中，如果我们有一些具体的东西，具体的规范和具体的要求要遵守，那么我们就需要让agent知道它到底要干啥。

如果他不知道，那么这些微小的决策和假设累积出来，就会让他越搞越不知道该怎么搞，这本质上还是个沟通问题。

现在解决人对agent的沟通问题的方式和研究有很多，比如spec驱动的开发啊，啥的，很多。

可能有的人也不研究这些，只是一味的用自然语言跟ai解释，我要啥，或者礼貌的耐心的讲我要啥，主要还是vibe coding为主。

在我看来，所谓的vibe coding很大程度是一种微管理。但是和你去管理人不同，agent作为一种极为高效的执行者，去微管理这个东西实际上是非常让人疲劳的，因为你需要不断的快速的去做决策，而做决策之前，你还得明白agent写了些什么屎.

这就跟餐厅的经理不去管绩效，非要去盯着那个刷碗的人教他怎么刷碗一样。

其实这个事本质上跟带人差不多，与其关注于ai做的每个决策是不是正确，不如关注ai所做决策后的让他去做这样决策的链条，和他做决策之后，能不能有个持续的机制去反馈让他自己知道决策的正确与否。

其实就我的经验来讲，很多现代的coding agent已经强大到可以自己跑很久，去解决很多具体的问题，去自己获得自己所需要的反馈了。

只有这样我们才能真正的成倍的，让人顶得住的，可持续的解放生产力。

当coding agent在跑的时候，去刷碗，拖地，找同事聊聊天啥的吧，我现在一般都桌子旁边放俩哑铃，他在跑我就在旁边活动活动。这些是agent做不了的事。

不要充满焦虑和期待的盯着屏幕，如果你早已选择了bypass approval，那么你再怎么盯也阻止不了它删库跑路，哈哈哈哈

有人看我就细说，我在不看着ai干活这件事情上具体做了哪些事情，来让ai在不被盯着一步步执行的情况下，能在真正生产级的环境中工作的。
