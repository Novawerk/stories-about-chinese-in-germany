# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository nature

A **Next.js 16 magazine-style site** whose content is hand-written Chinese memoirs. Two layers:

1. **Content**: Markdown files live at the repo root under folders whose names start with `🇩🇪`. Each folder is a *collection*. Chinese + emoji in folder/filenames — always quote them in shell commands.
2. **Site**: Next.js App Router under `src/`, Tailwind CSS, static-prerendered. Deployed to Vercel at **stories.novawerk.io** (project: `novawerk/stories-about-chinese-in-germany`, connected to this GitHub repo).

Replies to the user should generally be in Chinese unless they write in English.

## Content → site pipeline

```
🇩🇪<collection>/<chapter>.md     (source of truth, hand-edited)
          │
          │  scripts/sync-content.mjs   (runs at prebuild/predev)
          ▼
content/<slug>/<post-slug>.md    (git-ignored, consumed by Next.js)
content/<slug>/index.json
content/collections.json
```

- **Source**: any top-level `🇩🇪*` folder is discovered automatically — add a new collection by making a new folder, no code change needed.
- **Skipped files**: `完整故事.md` and `修改意见.md` are still in the ignore list as a safety net (they were artifacts of an older merge-chapters pipeline that's been removed) — don't recreate them.
- **Per-collection config**: optional `meta.json` in the folder overrides `slug` / `title` / `subtitle` / `tagline` / `order`. Defaults for the original three collections are baked into `scripts/sync-content.mjs` — move them to per-folder `meta.json` if the defaults ever get in the way.
- **Per-post config**: frontmatter at the top of a `.md` can override `title` / `order` / `excerpt` / `slug`. If no frontmatter is present, the script derives the title from (in order): a `# heading` line, the legacy `N/标题` first-line convention, or the filename stripped of its `N-` prefix.

The generated `content/` directory is `.gitignore`d.

## Reading view

Each collection renders as a **single long page** — `/<slug>` contains the hero, a Contents grid (目录), and all chapters stacked with a horizontal ornament between them. There are no per-chapter URLs; chapter-level deep links go via fragment, e.g. `/entrepreneur#引子`. Anchor IDs are the raw Chinese slugs — modern browsers handle them fine, they just URL-encode in the address bar.

## Day-to-day commands

```bash
pnpm install
pnpm dev          # auto-syncs content, serves http://localhost:3000
pnpm build        # auto-syncs content, full prod build
pnpm sync         # run only the content sync (rare)
```

A `.claude/launch.json` config is committed so `preview_start name="next-dev"` works out of the box inside Claude Code.

## Deployment

- **Production**: `git push` to `main` → Vercel builds → `stories.novawerk.io`.
- **Preview**: open a PR → Vercel posts a preview URL in the PR.
- **Vercel project**: `novawerk/stories-about-chinese-in-germany`, linked via `vercel git connect` to this repo. Domain `stories.novawerk.io` is a subdomain of `novawerk.io`, whose DNS Vercel already manages (nameservers `ns*.vercel-dns.com`), so no external DNS config is needed.
- First-time local setup uses `pnpm` (check lockfile); `vercel --scope novawerk` is the CLI context.

## Editing conventions

- **Chapter filenames**: `N-标题.md` — the `N` is the sort key; strip-accurate even if the file has no heading.
- **First line**: either `# 标题`, `N/标题` (legacy), or plain prose. All three parse correctly; the sync script also strips the title line from the body so it never renders twice.
- **New collection**: `mkdir "🇩🇪新栏目名"` + drop `.md` files inside. Optionally add `meta.json` for a nice slug/title. The homepage and navigation pick it up on next build.
- **Submissions**: `.github/ISSUE_TEMPLATE/new-story.yml` is the intake form; `/submit` is the on-site guide. Both point back to the repo.

## What NOT to touch without reason

- `content/` — it's regenerated from the 🇩🇪 folders on every build. Hand-edits are wiped.
- `完整故事.md` / `修改意见.md` — legacy artifacts; not in the site.
- `.vercel/` — project link metadata; regenerated by `vercel link`.
